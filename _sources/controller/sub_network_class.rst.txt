Subnetwork controller class
=====================================

Purpose
-------

This page documents the controller-side class used to represent and manage a
hydraulically connected subnetwork.

Use this reference for implementation details about how controller logic groups
assets and stores subnetwork-specific control state.

For user-facing explanation of how subnetworks affect simulation behavior,
start from :doc:`controller`.
